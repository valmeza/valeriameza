## Why

The current opportunity is to present the portfolio as a technical artifact, not a conventional marketing site. A terminal-inspired experience makes the site feel aligned with the audience and keeps the first version fast, focused, and easy to extend.

## What Changes

- Create a static Astro site for the portfolio.
- Replace traditional hero styling with a terminal-like header and command-prompt presentation.
- Present About, Skills, and Projects as compact, data-dense sections with TUI borders and monospace typography.
- Add keyboard-oriented navigation cues and hover states that fit a terminal workflow.
- Add GitHub Pages deployment for the static build.
- Enforce a dark, high-contrast TUI color palette across the site.

## Capabilities

### New Capabilities
- `terminal-portfolio-experience`: terminal-style hero, section layout, keyboard cues, responsive TUI presentation, and project showcase behavior.
- `static-github-pages-delivery`: static build, deployment configuration, and publish flow for GitHub Pages.

### Modified Capabilities

## Impact

Affected code includes the Astro app structure, Tailwind configuration, shared layout and section components, and deployment workflow files for GitHub Pages.
